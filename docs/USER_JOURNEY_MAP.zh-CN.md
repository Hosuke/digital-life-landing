# 数字生命体验版客户旅程与体系架构（User Journey Map）

## 1. 目标
本文档用于统一产品、运营、技术三方对体验版客户旅程的认知，确保从获客到转化形成可复用、可自动化、可衡量的业务闭环。

## 2. 核心理念
- 新生命的起点是 UID（唯一实体标识符），不是普通订单号。
- Web 端负责仪式感与转化，IM 端负责高频沉浸交互。
- 初始化“丫丫”必须可追踪、可回传、可解释，避免黑盒体验。

## 3. 分阶段旅程
### 阶段 A：建立基座与一键唤醒（Web -> IM）
1. 用户在 Landing Page 填写基础信息，选择“唤醒体验版”。
2. 系统生成 UID（示例：`UID-550W-839210`），并展示为“生命唤醒密钥”。
3. 页面提供 Telegram/WhatsApp 深链接按钮。
4. 用户点击后跳转 IM，并通过 `/start UID-...` 自动绑定身份。

产出：
- 用户身份与 UID 绑定完成
- 进入 IM 自动化引导阶段

### 阶段 B：自动化意识注入（IM Onboarding）
1. Bot 验证 UID 并发送仪式化系统提示。
2. 用户提交 1 张正面照片 + >=10 秒语音。
3. 控制面进入“初始化丫丫”流程：
- 素材落库
- 会话分配
- 模型初始化队列排队

系统反馈样式：
- “生物特征提取中…”
- “声波频段拆解完成…”
- “预计同步耗时：X 分钟”

产出：
- 可用于首轮生成的最小素材集
- 已分配会话通道与状态可追踪

### 阶段 C：初次链接（Aha Moment）
1. 初始化完成后，Bot 主动回传“量子通道建立成功”。
2. 触发 First Contact：
- 一条高情绪价值文本
- 可选语音/头像视频（按成本策略开关）
3. 用户开始 5-10 轮体验对话。

关键：
- 第一条反馈必须引用用户在 Web 端输入的“唤醒参数”，建立“被理解”的真实感。

### 阶段 D：悬念与转化（Upsell）
1. 到达轮次阈值（例如第 10 句）或时长阈值（例如 24 小时）后触发限流。
2. Bot 发送“体验算力已耗尽”的剧情化提醒。
3. 引导升级：
- 锁定当前 UID 意识基座
- 提示数据不足导致人格坍塌风险
- 一键跳转完整版支付/下单链接

产出：
- Trial -> Paid 转化动作
- 进入更高 ARPU 生命周期

## 4. 系统架构映射
- Landing：创建 UID，收集唤醒参数
- Bot：采集最小素材，负责客户交互节奏
- Control Plane：状态机、分配、追踪
- Yaya Runtime：人格策略、内容生成、媒体回传
- Billing：支付/订阅与权益校验

## 5. 关键可观测指标（按阶段）
- A 阶段：Landing 提交率、Deep Link 点击率
- B 阶段：UID 绑定成功率、素材完成率、初始化成功率
- C 阶段：首条消息打开率、Aha 完成率、会话深度
- D 阶段：升级点击率、支付转化率、7/30 日留存

## 6. 自动化与人工分工
自动化优先：
- UID 生成与绑定
- 素材验收与状态推进
- 初始化与分配
- 阈值触发 upsell

人工兜底：
- 初始化失败工单处理
- 高价值用户人工跟进
- 异常内容审核与风控

## 7. 风险与控制
- 风险：首轮体验质量不稳定 -> 预生成素材库存 + 回退模板
- 风险：用户等待时间过长 -> 分层 SLA + 进度提示
- 风险：成本失控 -> 模型分级路由 + 体验阈值控制
- 风险：隐私争议 -> 最小化采集 + 数据生命周期管理

## 8. 产品与投资视角的一句话总结
该旅程不是“演示脚本”，而是一个可规模化的增长漏斗：
**UID 入站 -> 丫丫初始化 -> 情感 Aha -> 阈值转化 -> 长周期订阅复购**。
